# FlowNana UI Review Checklist

本清单用于提测前快速自检，确保页面符合当前项目设计规则（Stone/Zinc + shadcn + v0 集成规范）。

## 0) 快速使用方式

- 适用范围：`/app` 页面、`/components/blocks` 业务模块、`/components/ui` 原子组件变更。
- 推荐节奏：开发完成后自检一遍；提测前再过一遍。
- 验收标准：每项都应为 `PASS`；如 `FAIL` 需给出修复备注。

---

## 1) 视觉基调与颜色

- [ ] 主背景是否为温润浅色（如 `bg-[#FDFDF9]` / `bg-stone-50`）？
- [ ] 主标题是否使用 `text-stone-900`？
- [ ] 正文是否使用 `text-stone-600`，辅助文字是否更浅（如 `text-stone-500/400`）？
- [ ] 是否已避免使用 `Slate/Gray` 作为主色体系？
- [ ] 关键强调是否使用克制暗色（如 `bg-stone-800`），而非亮蓝高饱和色？

## 2) 圆角、边框、阴影

- [ ] 卡片/容器圆角是否统一为 `rounded-xl` / `rounded-2xl`？
- [ ] 边框是否轻量柔和（如 `border border-stone-200/50`）？
- [ ] 默认阴影是否为 `shadow-sm`（无厚重黑影）？
- [ ] 浮层/弹窗是否使用柔和层次（如 `shadow-lg` + 浅色阴影）？

## 3) 排版与留白

- [ ] 页面是否有足够呼吸感（容器常见 `px-6 py-8` 或 `p-8`）？
- [ ] 区块间距是否统一（优先 `gap-6/gap-8`）？
- [ ] 文本是否具备可读行高（如 `leading-relaxed`）？
- [ ] 段落和模块是否避免拥挤（如 `space-y-4`）？

## 4) 交互与动效

- [ ] 可点击元素是否统一 `transition-all duration-300`？
- [ ] 按钮是否具备明确 Hover/Active（如 `hover:bg-stone-800/90 active:scale-[0.98]`）？
- [ ] 卡片 Hover 是否为“轻微”变化（微亮/轻阴影），无夸张动画？
- [ ] Focus 态是否可见（键盘可访问）？

## 5) 组件与目录分层

- [ ] 基础控件是否优先复用 `@/components/ui/*`？
- [ ] 复杂业务模块是否放在 `@/components/blocks/*`？
- [ ] 页面文件 `@/app/*` 是否以“数据获取 + 模块拼装”为主？
- [ ] 页面级 UI 逻辑是否避免过长（目标不超过 150 行复杂逻辑）？
- [ ] 图标是否统一使用 `lucide-react`，尺寸优先 `w-4 h-4` / `w-5 h-5`？

## 6) 响应式与布局安全

- [ ] 是否按 Mobile-First 实现（`sm:` / `md:` / `lg:`）？
- [ ] 是否避免固定像素宽度（如 `w-[800px]`）？
- [ ] 是否改用相对宽度（如 `w-full max-w-*`）？
- [ ] 在小屏设备下，关键 CTA、标题、表单均可完整显示？

## 7) 工程约束

- [ ] 是否完全避免内联样式（禁止 `style={{ ... }}`）？
- [ ] 重复样式模式是否已抽成组件/变体？
- [ ] Token/主题语义是否落在全局变量或统一语义类中？

## 8) 状态与可用性

- [ ] 是否有空态（Empty）？
- [ ] 是否有加载态（优先 Skeleton）？
- [ ] 是否有错误态（Error）？
- [ ] 是否有禁用态（Disabled）？
- [ ] 关键操作是否提供反馈（成功/失败/重试）？

## 9) 最终验收（提测前 1 分钟）

- [ ] 视觉风格一致（同一页面无“多套设计语言”混用）  
- [ ] 颜色、圆角、边框、动效符合统一规范  
- [ ] 关键流程在移动端可用  
- [ ] 无明显可访问性问题（焦点、对比度、可点击区域）  
- [ ] 无明显工程违例（内联样式、固定宽度、重复代码未抽离）
